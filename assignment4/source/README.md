# 二维 eVTOL 起降点选址与路径规划可视化程序

本项目是一个面向论文演示、组会展示和算法扩展的二维 eVTOL 可视化系统，包含两个连续模块：

```text
第一模块：需求点生成 -> K-means 候选点生成 -> GA 综合优化选址 -> 选址结果可视化
第二模块：选择客户起降点 -> 机场接驳终点 -> A* / RRT* 路径规划 -> 动态过程与指标对比
```

## 3. 项目结构

- `main.py`：Streamlit 主界面，包含选址模块和路径规划模块。
- `evtol/scene.py`：生成二维城市区域 `[0,100] x [0,100]` 内的旅客需求点。
- `evtol/fields.py`：生成连续风险场和成本场。
- `evtol/clustering.py`：内置 NumPy 加权 K-means，生成候选起降点。
- `evtol/ga.py`：遗传算法，从候选点中选择最终起降点。
- `evtol/visualization.py`：第一模块的 Plotly 可视化。
- `evtol/path_planning.py`：A* 与 RRT* 路径规划算法。
- `evtol/path_analysis.py`：路径长度、风险、成本、曲折度等指标分析。
- `evtol/path_visualization.py`：第二模块的动态搜索过程和路径对比可视化。
- `requirements.txt`：项目依赖。
- `run_app.bat`：Windows 一键启动脚本。

## 4. 第一模块：起降点选址

第一模块用于生成城市低空交通场景并完成起降点选址。

主要功能：

- 随机生成旅客需求点，可通过多个高斯需求热点模拟真实城市需求。
- 生成风险场，例如禁飞区、人口密集区、重要建筑区等。
- 生成成本场，例如中心城区高成本、交通走廊低成本、多中心商务区高成本。
- 使用 K-means 对需求点聚类，聚类中心作为候选起降点。
- 使用 GA 从候选点中选择最终起降点。
- 展示需求点、风险/成本热力图、候选点、最终选址点和 GA 收敛曲线。

选址目标函数：

```text
F = w1 * 需求覆盖率 - w2 * 平均风险 - w3 * 平均成本
```

## 5. 第二模块：路径规划与算法对比

第二模块直接复用第一模块输出的 GA 最终起降点。

完整流程：

```text
选择客户起降点 -> 固定机场接驳终点 -> 设置路径代价权重 -> 运行 A* / RRT* -> 查看动态搜索过程 -> 指标对比
```

### 起点与终点

- 起点：用户从 GA 最终选中的客户侧起降点中选择。
- 终点：固定机场接驳点，默认坐标为 `(92, 12)`。

### A* 算法

A* 在二维网格环境中运行：

- 将场景离散化为网格。
- 风险高于阈值的区域视为障碍/禁飞区。
- 支持障碍膨胀半径，用于表达安全距离。
- 支持欧氏距离或曼哈顿距离启发函数。
- open list / closed list 会在图中以不同颜色显示。

A* 代价函数：

```text
step_cost = α * distance + β * risk + γ * cost
f(n) = g(n) + h(n)
```

### RRT* 算法

RRT* 在二维连续空间中运行：

- 随机采样并扩展搜索树。
- 支持目标偏置概率，提高向终点收敛的速度。
- 支持障碍碰撞检测。
- 支持邻域重连，体现 RRT* 的渐近优化思想。
- 支持路径平滑处理。

RRT* 边代价：

```text
J = α * L + β * R + γ * C
```

其中 `L` 是边长，`R` 是边经过区域的平均风险累计，`C` 是边经过区域的平均成本累计。

## 6. 第二模块参数说明

通用参数：

- `路径长度权重 α`：越大越偏向短路径。
- `风险权重 β`：越大越倾向绕开高风险区域。
- `成本权重 γ`：越大越倾向经过低成本区域。
- `风险障碍阈值`：风险超过该值的区域被视为不可通行障碍。

A* 参数：

- `网格分辨率`：网格越细，路径越精细，但计算量更大。
- `启发函数`：可选欧氏距离或曼哈顿距离。
- `障碍膨胀半径`：以网格为单位扩大障碍区域，用于模拟安全距离。

RRT* 参数：

- `最大迭代次数`：采样和扩展次数。
- `步长`：每次扩展的最大距离。
- `邻域半径`：用于选择更优父节点和重连。
- `目标偏置概率`：采样时直接采到目标点的概率。
- `障碍安全缓冲`：碰撞检测采样间隔相关参数。
- `启用 RRT* 路径平滑`：对最终路径做快捷连线平滑。

动态显示：

- `动态搜索进度`：控制显示搜索过程的前多少比例。拖动滑块可模拟逐步搜索动画。

## 7. 量化分析指标

第二模块会输出以下指标：

- `path_length`：总路径长度。
- `cumulative_risk`：路径累计风险。
- `cumulative_cost`：路径累计成本。
- `average_risk`：路径平均风险。
- `min_obstacle_distance`：路径到障碍区域的近似最小距离。
- `runtime_s`：算法计算时间。
- `node_count`：A* 已扩展节点数或 RRT* 树节点数。
- `turn_count`：路径转折次数。
- `tortuosity`：曲折度，等于路径长度 / 起终点直线距离。
- `success`：是否成功找到可行路径。

## 8. 对比与导出

程序支持：

- 单独运行 A*。
- 单独运行 RRT*。
- 同时运行 A* 和 RRT* 并叠加显示路径。
- 用表格展示两种算法的量化指标。
- 用柱状图比较路径长度、累计风险、累计成本、计算时间、转折次数和曲折度。
- 一键运行所有最终起降点到机场终点的批量路径对比。
- 导出当前路径指标 CSV。
- 导出所有起点路径对比 CSV。
- 保存当前路径图为交互式 HTML。

## 9. 后续扩展建议

可以继续扩展以下方向：

- 将 RRT* 替换或补充为 Informed RRT*、BIT*、PRM、D* Lite。
- 加入真实禁飞区、多边形障碍和城市建筑物轮廓。
- 加入风场、航路容量、转弯半径、最大航程和爬升下降约束。
- 加入多目标 Pareto 前沿，展示风险、成本、时间之间的权衡。
- 将机场接驳终点扩展为多个机场或枢纽点并自动分配。
